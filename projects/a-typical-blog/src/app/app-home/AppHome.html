New Concepts

Primary Goals

  1. Create a blog site that is compatible with Angular SSR
  2. Keep components lightweight and as declarative as possible
  3. Strict validation on the server side

The Stack

Frontend

  • Angular 6
  • Angular Material
  • RxJS 6
  • Other libraries
    • Quill

Backend

  • Node.js 8
  • Koa 2
  • Angular 6 (server platform)
  • SocketCluster
  • Passport.js
  • MongoDB

Problems encountered along the way

  • Angular 6 is still in RC phase, so I did have to trace several bugs and work around them
  • Chrome bug with position: sticky!
  • More Chrome bugs related to how border-radius impacts the visible width of borders on windows with non-integer pixel ratios
  • Quill attempts to manipulate the document upon module load causing problems for SSR

Features that were planned but didn't make it because I got tired of focusing on the frontend

  • Unit testing for Angular components
  • UI for creating replies as children of comments (YouTube-style)
  • Comment voting
